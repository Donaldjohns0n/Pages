# Curated Open Source Accounting Frameworks

The following catalog highlights open source projects suitable for AI-enhanced accounting automation. Scores are derived from the repository tracking system and the evaluation workbench rubrics.

| Project | Primary Focus | AI/Automation Hooks | License | Community Health | Recommended Use Cases |
|---------|---------------|---------------------|---------|------------------|-----------------------|
| **ERPNext** | Full-stack ERP with accounting module | Machine learning extensions via Frappe apps; workflow automations | GPL v3 | High (frequent releases, >2k contributors) | Mid-market finance teams seeking customizable workflows and integrated reporting |
| **Odoo Accounting (Community Edition)** | Modular ERP with strong accounting support | Studio automation, AI-driven invoice digitization add-ons | LGPL v3 | High (active partner network, large module ecosystem) | Organizations needing modular rollouts with optional AI features |
| **LedgerSMB** | Double-entry accounting and invoicing | API endpoints ready for AI classification services | GPL v2 | Medium (steady maintainer activity) | SMEs requiring auditable, PostgreSQL-backed accounting |
| **Akaunting** | Cloud-first accounting for SMEs | REST APIs and webhook support for AI transaction tagging | GPL v3 | Medium (growing plugin marketplace) | Freelancers and small businesses exploring AI-assisted bookkeeping |
| **Firefly III** | Personal/SMB finance management | ML plugins for transaction categorization; budgeting automation | AGPL v3 | Medium (vibrant community discussions) | Finance teams prototyping AI categorization workflows |
| **Frappe Insights** | Analytics layer for Frappe/ERPNext data | Supports custom ML models for anomaly detection and forecasting | MIT | Emerging (new but active roadmap) | Teams seeking rapid analytics over accounting data |

## Selection Guidance
1. Match license obligations with organizational compliance requirements.
2. Prioritize projects with active governance and clear release roadmaps.
3. Leverage the repository tracking system for detailed maturity and risk scores.
4. Use sandbox runbooks to validate AI integrations before production deployment.
